# 8zz 巴逆逆反指標 - 完整開發規格書

## 專案概述

建立一個公開的「8zz 巴逆逆反指標」產品，包含：
1. **升級版 TradingView Pine Script 指標**
2. **GitHub Pages Landing Page（新 repo）**
3. **自動更新管線**（banini-tracker → Pine Script 自動生成）

背景：巴逆逆（8zz）是台灣股市知名的「反指標女神」，她買什麼跌什麼、賣什麼漲什麼。本專案將她的 FB 貼文操作記錄轉換成 TradingView 指標，讓用戶在 K 線圖上驗證反指標效果。

## Part 1：升級版 Pine Script 指標

### 1.1 現有基礎（已完成，直接延續）

目前已有可運作的 Pine Script v6 指標，功能：
- 33 筆 FB 真實事件（2026/02-04），hardcoded Unix 毫秒時間戳
- 方向翻轉時顯示大箭頭（▲ 偏多綠色 #00E676 / ▼ 偏空紅色 #FF1744）
- 同方向訊號可選擇顯示小圓點（預設關閉）
- 右上角資訊面板：方向、倉位水位（x/5, x*20%）、勝率、觀察窗
- hover tooltip 顯示詳細資訊（她的動作、標的、FB 發文時間、原文摘要）
- `indicator()` 模式（不是 strategy），避免 TradingView 自動交易標籤污染畫面
- 箭頭距離 K 線可調整（offset_pct 參數）

### 1.2 需要新增的功能

#### A. 訊號強度分級（1~3 級）
- 根據巴逆逆的用語情緒強度分級：
  - **強度 3（強）**：all in、停損、畢業、認賠、虧損 X 位數、漲停跌停
  - **強度 2（中）**：買進、加碼、賣出、減碼、被套
  - **強度 1（弱）**：觀望、考慮、持有、看多看空但未行動
- 箭頭大小對應：強=size.huge、中=size.large、弱=size.normal
- tooltip 中顯示「強度: ★★★」或「強度: ★★☆」或「強度: ★☆☆」
- 每筆事件資料需要新增 `evt_strength` 陣列

#### B. 背景色帶
- 偏多期間：淡綠背景 `bgcolor(color.new(#00E676, 95))`
- 偏空期間：淡紅背景 `bgcolor(color.new(#FF1744, 95))`
- 無訊號時：透明
- 需要一個開關「顯示背景色帶」（預設開）

#### C. 訊號存活指示
- 在最新一筆翻轉箭頭的 tooltip 中加入「距發文已過 X 小時」
- 右上面板也顯示「訊號年齡: X 小時前」
- 超過 48 小時的訊號在面板中標示為「已過期」灰色

#### D. 累計績效曲線（獨立面板）
- 這個比較複雜，可以做成獨立的第二個指標（因為 overlay=true 和 overlay=false 不能共存）
- 或者用 `plot()` 在主圖下方畫一條線（但會很小）
- 建議：做成第二份 Pine Script，名為「8zz 績效追蹤」，overlay=false
- 顯示：每次翻轉時記錄價格差，累加成一條損益曲線
- 加上零線 hline(0)
- 顏色：正值綠、負值紅

#### E. 右上面板增強
- 新增一行「她目前持有」：顯示最近一筆偏空訊號中提到的標的名稱
- 新增「最近動作」：最近一筆事件的簡短描述（例如「4/10 買台光電」）
- 面板配色更精緻，標題行用深灰底

#### F. 開關系統（在「輸入」分頁）
```
顯示分組：
  - 箭頭距離 % (float, 預設 2.0)
  - 顯示加倉圓點 (bool, 預設 false)
  - 顯示背景色帶 (bool, 預設 true)
  - 顯示資訊面板 (bool, 預設 true)

回測分組：
  - 觀察K棒數 (int, 預設 5)
```

#### G. 可見性設定提醒
- 在指標描述文字中註明：「建議在可見性中取消勾選日、週、月，只保留秒、分鐘、小時，以確保指標精確度」
- Pine Script 本身無法控制可見性，這是用戶手動設定

### 1.3 交易邏輯（寫在指標描述和 tooltip 中）

```
操作邏輯：
1. 巴逆逆買進/加碼/被套/看多 = 指標偏空 ▼
2. 巴逆逆停損/賣出/畢業/認賠 = 指標偏多 ▲
3. 每出現同方向訊號加倉 20%，最多 5 次 (100%)
4. 出現反轉訊號 → 立刻全部平倉 → 反手建倉 20%
5. 指標有效期約 1~3 天（根據回測，可在設定中調整觀察窗）
```

### 1.4 事件資料格式

每筆事件有 4 個陣列同步 push：
```
evt_time:     Unix 毫秒時間戳 (int)
evt_dir:      方向 (int, 1=偏多, -1=偏空)
evt_strength: 強度 (int, 1=弱, 2=中, 3=強)
evt_tips:     tooltip 文字 (string, 用 \n 換行)
```

## Part 3：自動更新管線

### 3.1 generate-pine.js 腳本

放在 banini-tracker repo 中，功能：

1. 讀取 `data/report-*.json`（banini-tracker 產出的分析結果）
2. 或讀取手動建立的 `events.json`
3. 解析每筆事件：時間、動作、標的、信心、原文摘要
4. 自動判定方向（買=偏空、賣=偏多）和強度（1~3）
5. 計算 Unix 毫秒時間戳
6. 輸出完整的 Pine Script 檔案

用法：
```bash
# 從 banini-tracker 的 report JSON 生成
node generate-pine.js

# 從手動 events.json 生成
node generate-pine.js --json events.json

# 指定輸出路徑
node generate-pine.js --output ../8zz-indicator/indicator.pine
```

### 3.2 未來自動化流程（Phase 2）

```
banini-tracker 抓到新貼文
  → AI 分析動作和方向
  → 寫入 report JSON
  → 觸發 generate-pine.js
  → 新版 Pine Script 推送到 8zz-indicator repo
  → GitHub Pages 自動更新
  → 用戶從網站複製最新版
```

## 注意事項

1. Pine Script 中絕對不要用 `strategy()` 模式，只用 `indicator()`，避免 TradingView 自動交易標籤污染畫面
2. 箭頭用 `label.new()` + 透明背景 + 文字三角形（▲▼），不用 `plotshape()`
3. tooltip 中的換行用 `\n`，不能有跨行字串
4. 所有時間戳使用 Unix 毫秒（UTC），不要用 `timestamp()` 函數避免時區問題
5. 顏色固定：偏多綠 #00E676、偏空紅 #FF1744
6. Landing page 中的免責聲明要明顯，不能藏在角落
7. 繁體中文，不使用「您」敬稱，用平常語氣
8. 不使用「——」全形破折號
